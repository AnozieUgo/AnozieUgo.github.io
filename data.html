Data content and source

To use any of these API links, an API-Key is needed which would be added to the header of the request.


Top Four Most Populous Countries in Asia

The data was sourced from the live api https://get-population.p.rapidapi.com/population/country?country=${country}. The API fetches the world population in realtime. However, using this wikipedia resource https://en.wikipedia.org/wiki/List_of_Asian_countries_by_population, I sourced for the four most populous asian countries and serve each of them to the api at ${country}.

Thus, having an api link of this nature; https://get-population.p.rapidapi.com/population/country?country=${country}.

Nature of request:
To access the data, make an api-request using the format below; 

                    const url = 'https://get-population.p.rapidapi.com/population/country?country=China';
                    const options = {
                        method: 'GET',
                        headers: {
                            'X-RapidAPI-Key': 'YOUR-API-KEY',
                            'X-RapidAPI-Host': 'get-population.p.rapidapi.com'
                        }
                    };
The structure of the response data is as follows; 
                    {
                        count:1456820983
                        readable_format:"1,456,820,983"
                        country:"China"
                      }
However, for this project, I only picked the readable_format key of the object

Covid-19 Effect in the top four most asian populated countries

The data was sourced from the live api https://covid-19-coronavirus-statistics.p.rapidapi.com/v1/total?country=${country}, where ${country}, is the placeholder for any of the countries (China, India, Parkistan, Indonesia).

The API fetches the covid-19 reports of different countries. However, using this wikipedia resource https://en.wikipedia.org/wiki/List_of_Asian_countries_by_population, I sourced for the four most populous asian countries and serve each of them to the api at ${country}.

Thus, having an api link of this nature; https://covid-19-coronavirus-statistics.p.rapidapi.com/v1/total?country={country}

Nature of request:
To access the data, make an api-request using the format below; 

                    const url = `https://covid-19-coronavirus-statistics.p.rapidapi.com/v1/total?country=${country}`;
                    const options = {
                        method: "GET",
                        headers: {
                        "X-RapidAPI-Key":
                            "YOUR-API-KEY",
                        "X-RapidAPI-Host": "get-population.p.rapidapi.com",
                        },
                    };
The structure of the response data is as follows; 
                    {
                        "error": false,
                        statusCode:200
                        message:"OK"
                        data:
                        recovered:null
                        deaths:101056
                        confirmed:4903524
                        lastChecked:"2023-08-18T13:48:46+00:00"
                        lastReported:"2023-03-10T04:21:03+00:00"
                        location:"China"
                        }
                      }
However, for this project, we are only picking the deaths and confirmed keys of the data object

Description

Data virtualization is a technology that enables efficient and seamless access to dispersed data sources, regardless of their physical location or structure. By providing a unified and abstracted view of data, it minimizes data duplication and reduces the need for complex ETL processes. This approach enhances agility and simplifies data integration for improved decision-making across organizations.
